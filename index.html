DAU vs DUU — Correlation, Ratio & Seasonality

r = 0.98 · N = 415 days

Daily Active Users (DAU) vs Daily Unique Users (DUU) — Jan 2025 to Mar 2026

DAU counts users who took an intentional action; DUU counts any unique visitor including background sync and passive sessions. Despite measuring different things, the two track almost perfectly (r = 0.98) — DUU runs consistently ~13–14× higher than DAU. Both metrics show the same seasonal arc: a steady climb from Jan → May 2025, a summer plateau (Jun–Aug), a recovery in Sep–Nov, and a pronounced holiday dip in Dec–Jan.

DAU / DUU Ratio (%) — Weekday vs Weekend & Seasonal Pattern

The ratio separates cleanly into two regimes: weekdays ~7.3% and weekends ~4.0% — a drop of ~45%. DUU stays elevated on weekends (background sync, mobile, consumer use) while DAU collapses, confirming DAU is the sharper enterprise-work signal. Seasonal compression: the weekday ratio drifts upward from ~7.2% in Jan 2025 to ~7.5–8.0% by Apr–May 2025, then retreats over summer. Holiday periods (Dec 25 – Jan 4) see the ratio fall to 3–4% as Enterprise activity disappears while consumer DUU persists.

Monthly Average DAU vs DUU — Seasonal Trend

Both metrics grow through early 2025 then plateau. The summer dip (Jul–Aug) is visible in DAU but barely registers in DUU — further evidence that DUU captures persistent background activity that doesn't vary with human work schedules.

DAU/DUU Ratio Distribution — Weekday vs Weekend

Weekday ratios cluster tightly around 6.9–8.5% with a long right tail in spring/summer peaks. Weekend ratios form a distinct lower cluster at 3.4–4.9%. The bimodal separation is clean — day-of-week is the dominant driver of the ratio, stronger than season or segment mix.
02

DAU & DUU as Predictors for Database Cores (MaxDbCores)

Best: r = 0.52 (farm-level DAU)

All User Signals vs MaxDbCores — Ranked by Correlation Strength

All-segment DAU (farm-level) and Consumer DUU (date-level) are the two best proxies for provisioned capacity, both at r ≈ 0.50–0.52. Isolating Enterprise — whether via DAU or DUU — makes the correlation worse, despite Enterprise being the largest segment (~40% of DAU, ~73% of DUU volume). Enterprise capacity is decoupled from user activity: it is driven by SLA contracts, compliance requirements, and peak-headroom buffers.

DAU vs MaxDbCores — Pearson r by Customer Segment

Government (r = −0.10) is effectively zero/negative — capacity and usage move independently. Enterprise (r = 0.22) is only marginally better. Viral & Other (r = 0.62) and Charity (r = 0.58) have the cleanest signal because these farms were sized closer to observed demand rather than contractual guarantees.

Scale Efficiency — MaxCores per M DAU by Farm Size Quintile

Larger farms are substantially more efficient: the largest farms (Q5, avg 173M DAU) require 55% fewer max cores per million users than the smallest (Q1, avg 25M DAU). This sub-linear scaling is why DAU alone is a moderate predictor — the DAU→cores relationship is non-linear and needs farm size as a co-variate for accurate forecasting.

DAU vs MaxDbCores — Farm Count & Avg DAU per Segment

Enterprise dominates by avg DAU (40M per farm) but has the second-weakest correlation. Education (20M avg DAU) and SM&C Commercial (12M) have meaningfully stronger signals despite lower volumes.

DUU Breakdown — Enterprise vs Consumer vs Total → MaxDbCores

Consumer DUU (r = 0.50) is nearly twice as predictive as Enterprise DUU (r = 0.28), even though Enterprise makes up 72.6% of total DUU volume. Total DUU (r = 0.33) sits between the two — its correlation is dragged down by Enterprise's dominant but noisy volume share.
03

File Count as a Predictor for DB / SQL Storage

r = 0.87 · N = 636 farms

File Count vs DB Storage — Per Farm (636 farms)  r = 0.87

File count explains 76% of DB storage variance across 636 farms (r = 0.87). The relationship is strong but not as tight as user storage (r = 0.93) because SQL/DB storage accumulates additional overhead beyond raw file bytes: metadata, permissions, sharing records, version history, and tenant configuration. Farms with unusually high DB storage relative to file count tend to be heavy collaboration hubs with complex permission trees and frequent versioning.
04

File Count as a Predictor for User Storage

r = 0.93 · N = 636 farms

File Count vs User Storage — Per Farm (636 farms)  r = 0.93

File count explains 87% of user storage variance — the strongest single-variable predictor available for storage capacity planning. User storage is essentially a direct byte-sum of file content, so the tighter fit vs DB storage is expected. Practical implication: file count growth rate is the most reliable leading indicator for storage provisioning decisions — more reliable than DAU, region, or segment mix alone.

Storage Predictor Comparison — DB vs User Storage (r values)

The 0.06 gap in r (0.93 vs 0.87) represents a meaningful difference in predictability. For storage capacity planning, always use user storage as the primary metric — it is cleaner, more directly file-driven, and requires fewer adjustment factors.

Top 20 Farms — File Count vs Both Storage Types

US farms dominate the top 20 by file count and both storage dimensions. The DB/User storage ratio varies across farms — higher ratios indicate heavier metadata load (collaboration-intensive tenants).
